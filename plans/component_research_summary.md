# Исследование компонентов для проекта настольных часов

## ESP8266

### Общая информация
ESP8266 - это микроконтроллер с встроенным WiFi модулем, популярный для IoT проектов. Для проекта рекомендуется использовать плату NodeMCU или Wemos D1 Mini из-за их удобства и наличия встроенного USB-интерфейса.

### Технические характеристики
- Процессор: 32-битный RISC CPU Xtensa LX106
- Тактовая частота: до 160 MHz
- RAM: 80 КБ
- Flash: 4 МБ (обычно)
- WiFi: 802.11 b/g/n
- GPIO: 17 пинов (на NodeMCU)
- Напряжение: 3.3V

### Рекомендации по использованию
- Использовать стабилизированный источник питания 3.3V
- Обязательно добавить конденсаторы фильтрации
- Учитывать ограничения по току на GPIO пинах (12 мА максимум)

## TCA9548A (I2C Multiplexer)

### Общая информация
TCA9548A - это 8-канальный I2C мультиплексор, который позволяет подключать несколько I2C устройств с одинаковыми адресами к одной шине.

### Технические характеристики
- 8 независимых I2C каналов
- Адреса: от 0x70 до 0x77
- Напряжение: 1.8V - 5.5V
- Частота шины: до 400 kHz (Standard/Fast mode)

### Принцип работы
- Каждый канал может быть включен/выключен программно
- Только один канал может быть активен одновременно
- Позволяет решить проблему конфликта адресов I2C устройств

### Рекомендации по использованию
- Подключать SDA и SCL от ESP8266 к основной шине мультиплексора
- Каждый выходной канал подключается к своим 8 SSD1306 дисплеям
- Использовать подтягивающие резисторы на каждом канале (4.7kΩ)

## SSD1306 (OLED дисплеи)

### Общая информация
SSD1306 - это OLED контроллер с разрешением 128x64 пикселей, популярный для Arduino проектов.

### Технические характеристики
- Разрешение: 128x64 пикселей
- Интерфейсы: I2C или SPI
- Напряжение: 3.3V или 5V (в зависимости от модуля)
- Адреса I2C: 0x3C или 0x3D

### Рекомендации по использованию
- Использовать I2C версию для экономии GPIO пинов
- Учитывать потребляемый ток (до 150 мА при максимальной яркости)
- При подключении через TCA9548A каждый дисплей будет иметь уникальный адрес

## APA106 (RGB светодиоды)

### Общая информация
APA106 - это адресные RGB светодиоды, аналогичные WS2812 и NeoPixels, но с более высоким напряжением питания.

### Технические характеристики
- Напряжение: 5V
- Цвет: RGB (24 бита на светодиод)
- Интерфейс: однопроводной (DIN/DOUT)
- Размер: 5050 (5.0x5.0 мм)

### Принцип работы
- Каждый светодиод имеет встроенный контроллер
- Светодиоды соединяются последовательно
- Информация передается в виде временных импульсов

### Рекомендации по использованию
- Использовать отдельный 5V питание для светодиодов
- Добавить резистор 300-500Ω между контроллером и первым светодиодом
- Использовать конденсаторы 1000 мкФ на питании
- Подключать через один GPIO от ESP8266

## Рекомендуемые библиотеки

### Для ESP8266
- ESP8266WiFi.h - для работы с WiFi
- ESP8266WebServer.h - для веб-сервера
- ESP8266mDNS.h - для mDNS (необязательно)

### Для I2C и TCA9548A
- Wire.h - стандартная библиотека для I2C
- Для мультиплексора потребуется создать специальную функцию для переключения каналов

### Для SSD1306 дисплеев
- Adafruit_SSD1306.h - основная библиотека для работы с дисплеем
- Adafruit_GFX.h - графическая библиотека
- Установить адреса дисплеев через мультиплексор (0x3C или 0x3D)

### Для APA106 светодиодов
- Adafruit_NeoPixel.h - библиотека для адресных светодиодов
- APA106 совместимы с NeoPixel, так что эта библиотека подойдет

## Архитектура подключения

```
ESP8266 (NodeMCU/Wemos D1 Mini)
├── VCC (3.3V) → TCA9548A VCC
├── GND → Все компоненты GND
├── SDA (GPIO4) → TCA9548A SDA
├── SCL (GPIO5) → TCA9548A SCL
├── GPIO (например GPIO14) → APA106 DIN
└── 5V Power → APA106 VCC (отдельный источник)

TCA9548A
├── Channel 0 → SSD1306 Display 1 (0x3C)
├── Channel 1 → SSD1306 Display 2 (0x3C)
├── ...
├── Channel 7 → SSD1306 Display 8 (0x3C)
```

## Потребляемая мощность
- ESP8266: до 240 мА при WiFi передаче
- Один SSD1306: до 20 мА
- 8 SSD1306: до 160 мА
- Один APA106: до 60 мА
- 8 APA106: до 480 мА
- Общий ток: до 880 мА при полной яркости

## Возможные проблемы и решения
1. Проблемы с питанием - использовать отдельный стабилизированный блок питания 5V
2. Проблемы с I2C шиной - использовать короткие провода и подтягивающие резисторы
3. Проблемы с памятью ESP8266 - оптимизировать использование памяти и использовать PROGMEM
4. Проблемы с синхронизацией - использовать таймеры и прерывания для синхронной работы
